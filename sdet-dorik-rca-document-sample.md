# How to File RCA (Root Cause Analysis)?

Source: [https://sdet.dorik.io/rca](https://sdet.dorik.io/rca)

## What is RCA?
Root cause analysis (RCA) is defined as a collective term that describes a wide range of approaches, tools, and techniques used to uncover causes of problems.

## Key Components to Fill in RCA
- **Description**: Add the full description, including what, how, and why this happened.
- **Detection**: How the issue was detected, by which team, and at what time.
- **What went Wrong**: Discussed with Dev/Ops – what exactly happened?
- **Correction**: What was the fix and when was it or will it be sent.
- **Prevention**: How it could have been avoided (e.g., missing test cases, automation, KT, better communication).
- **Impact**: Number of users impacted with data (provided by PM).
- **Timeline**: Enter logs from start to end with ETA.
- **Action Item**: Add test cases, monitoring emails, etc., with ETAs.
